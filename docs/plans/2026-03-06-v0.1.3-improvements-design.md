# ez v0.1.3 Improvements Design

**Date:** 2026-03-06
**Status:** Approved

## Problem Summary

Real-world usage of `ez` revealed documentation errors, missing UX polish, and several missing features. Issues were logged in `problems.md`. This doc captures the approved design for v0.1.3.

## Critical Bug

`README.md` incorrectly describes `ez push` as pushing "all branches in the stack" — it only pushes the current branch. The quick-start example is also misleading (shows `ez push` then `ez submit` as if separate steps).

## Changes

### `src/cli.rs`
- `Create`: add `-a`/`--all` flag (stage all tracked changes before committing)
- `Push`: add `--title`, `--body`, `--body-file`, `--base` flags
- `Submit`: add `--title`, `--body`, `--body-file` flags
- `Sync`: add `--dry-run` flag
- New `PrEdit` subcommand with `--title`, `--body`, `--body-file`

### `src/error.rs`
- `BranchAlreadyExists`: add actionable hint ("Use `ez checkout <name>` to switch")
- `NothingToCommit`: add actionable hints (stage first or omit `-m`)

### `src/git.rs`
- `push()`: detect "stale info" error, return a `StaleRemoteRef(String)` error variant
- Add `fetch_branch(remote, branch)` for targeted fetches before force-push

### `src/cmd/create.rs`
- Accept `all: bool` — call `git::add_all()` before staged-changes check

### `src/cmd/push.rs`
- Accept title, body, body_file, base options
- Auto-fetch branch before `--force-with-lease` to prevent stale ref errors
- Pass title/body/base through to `push_or_update_pr`

### `src/cmd/submit.rs`
- Accept title, body, body_file options — applied to newly created PRs

### `src/cmd/sync.rs`
- Add `dry_run: bool` — print planned actions without executing

### `src/cmd/pr_edit.rs` (new)
- Look up current branch's PR via `github::get_pr_status`
- Call `gh pr edit <number>` with provided `--title`, `--body`, `--body-file` args

### `src/github.rs`
- `create_pr`: accept `Option<&str>` title and body overrides
- Add `edit_pr(pr_number, title, body, body_file)` fn

### `README.md`
- Fix `ez push` description: current branch only, creates PR
- Fix quick-start: remove redundant `ez push` before `ez submit`
- Add clear table distinguishing `ez push` vs `ez submit`
- Document `ez create -m` requires staged changes; `-a` flag stages everything
- Document `ez pr edit`

### `~/.claude/CLAUDE.md`
- Fix `ez push` row description
- Add `ez pr edit` row

## Version
Bump `Cargo.toml` to `0.1.3`.
